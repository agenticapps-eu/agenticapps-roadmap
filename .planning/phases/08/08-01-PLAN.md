<objective>
Close CR-01's replay gap (D-08-06) before the write path first goes live: add a
Cloudflare KV binding to `wrangler.toml` and a consume-once nonce to the `apply`
branch of `functions/api/backfill/dispatch.ts`, so a reused `previewRunId` is
suppressed on a **best-effort sequential** basis. This is the only genuinely new
code in Phase 8.

Note on the guarantee: Workers KV is eventually consistent and offers no
compare-and-swap, so a `get(null) → put → dispatch` sequence is NOT atomic — a
duplicate request routed to a different edge location during the consistency
window (up to ~60s) can still pass. The property delivered here is therefore
"best-effort sequential replay suppression," not "exactly once." The 15-min
recency bound (unchanged) is the co-mitigation. This is the correct disposition
for the small trusted email allow-list (D-08-02) — CR-01's concern is naive,
in-window reuse, not a concurrent attacker. Do NOT escalate to a Durable
Object/D1 (out of scope per D-08-06 — single KV namespace, no new infra).

Purpose: A recent-but-reused preview can currently authorize multiple applies
within the 15-minute window; the nonce makes a `previewRunId` best-effort
single-use. Resolves `07-HUMAN-UAT.md` item #13.
Output: KV binding in `wrangler.toml`, check-then-set nonce in `dispatch.ts`, and
unit tests proving a sequential reuse is rejected without a second GitHub dispatch.
</objective>
