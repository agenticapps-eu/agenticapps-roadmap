## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public internet → Cloudflare Access → Pages | Access is the SOLE authn and (per D-07-03) the SOLE write authorization; everything behind it is trusted. Every reachable production hostname is an entry point. |
| Pages Function → GitHub Actions REST | dispatch/status carry the PAT (server-side only) to trigger + read backfill runs. |
| CI runner → sibling private repos + Linear | the PAT (Contents:Read) clones siblings; LINEAR_API_KEY writes Linear. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-06 | Spoofing | unauthenticated request to app root or any `/api/*` route bypassing the gate — including via an ALTERNATE production hostname (custom domain vs `<project>.pages.dev`) | mitigate | An Access application over the WHOLE domain (empty path) on EVERY reachable production hostname (D-08-01). Task 3 curls all four paths on each hostname (302/403) and verifies each 302's `Location` targets the Access login. A `200` from any unauth production path, or an ungated alternate hostname, FAILS the phase. |
| T-08-07 | Elevation of Privilege / Tampering | replay of a `previewRunId` to authorize an unreviewed apply (CR-01) | mitigate | 15-min recency bound (Phase 7) + KV best-effort sequential replay suppression (plan 08-01), verified live in Task 3 item #13 (reused id → 403). Duplicate remains possible only within KV's consistency window (accepted, T-08-03 in 08-01). |
