<objective>
Build the SYNC-04 write engine: apply.ts upserts one project's Linear records
(create-before-nothing: resolve, then create only what is missing) by executing the exact
operation set the diff approved, writes resolved ids back to linear-map.json atomically and
immediately after each create, and emits the planAhead flag into public/roadmap.json
through the existing leak/schema gate. Per the cross-AI review: v1 is create-only for
existing records (dates on existing milestones are informational, not written), apply
aborts on precondition drift (TOCTOU), the map write is atomic and per-create, and issue
descriptions come from the plan's taskLines. Per D-06-02, when a project is newly created
and its entry names an Initiative, apply attaches it via INITIATIVE_TO_PROJECT_CREATE using
the resolved Initiative ID (resolved.initiativeId). Dry-run must issue zero mutations; a
re-run must be a no-op. This plan is autonomous logic + mocked tests; the CLI wiring and
live human-verify are 06-07.

Output: apply.ts (+test: dry-run, idempotent, create-only, initiative-join, atomic map, planAhead — all against mocks).
