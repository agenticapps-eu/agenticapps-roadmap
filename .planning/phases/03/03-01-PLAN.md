<objective>
Make the Phase 02 query string and GQL→RawWorkspace mapping runtime-agnostic and shareable, and verify the Worker can import them — BEFORE the full handler is built.

Purpose: Both the Node sync script and the new Pages Function must reuse exactly ONE query and ONE mapping. Duplication is the precise leak vector the spec warns against. The mapping + its GraphQL types must live in a PROCESS-FREE module so the Worker (which gets a Worker-only tsconfig in 03-02) never transitively imports `client.ts`'s `process.env` access — otherwise the Worker typecheck breaks / Node globals leak into the Worker path (cross-AI review HIGH). This plan also de-risks Research Open Question #1 (cross-directory `.ts` imports under the Pages build) with a trivial probe handler, so the real handler in 03-03 builds on proven ground.
Output: `scripts/linear/query.ts`, a new process-free `scripts/linear/map.ts`, a refactored `client.ts` (unchanged Node behavior, imports from map.ts), a one-line guard in `transform.ts`, and a trivial probe handler at `functions/api/linear/[[path]].ts`.
</objective>
