<objective>
Re-run `pnpm sync:snapshot` so `public/roadmap.json` carries the real `project.url` values
added by the 04-01 pipeline, then commit the regenerated snapshot. This is the ONE step in
the phase that cannot run autonomously: `pnpm sync:snapshot` reads `process.env.LINEAR_API_KEY`,
which is currently UNSET (per STATE.md). The UI ships without it (the popover link is
gracefully omitted when `url` is null), so this plan can run any time after 04-01.

Dependency split (why this is wave 4, not wave 2): the snapshot regeneration itself only
has a HARD dependency on the 04-01 pipeline (which emits `url`) — that is the only entry in
`depends_on`. But this plan's end-to-end verification — opening a scheduled project's popover
and confirming the "Open in Linear ↗" link renders — requires the built Timeline UI
(04-04 popover body → 04-05 bar/pill trigger → 04-06 page). The plan is therefore placed in
wave 4 (run-last, alongside 04-06) as an ordering marker so the popover-link check is
meaningful, even though the snapshot write only needs 04-01. It remains a human-gated,
non-autonomous token step with graceful deferral (below).

Purpose: TL-03 — populate the authoritative Linear URL so the popover footer link renders.
Output: an updated, leak-gate-clean `public/roadmap.json` with `url` per project, committed.
</objective>
