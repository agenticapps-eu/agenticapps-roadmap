CLI truth table (scripts/sync-gsd-linear/cli.ts, invoked UNCHANGED — D-07-01):
`--project X` = dry-run preview (no roadmap.json write); `--project X --apply --yes` =
write, no prompt. NOTE: a real apply internally calls patchPlanAhead(..., true) writing
`planAhead: true` (apply.ts:414/431) — which SyncBadge renders as "Out of sync". That is
why the apply leg MUST run a full `pnpm sync:snapshot` AFTER the CLI: buildSnapshot
(transform.ts:147) rebuilds roadmap.json fully from Linear, includes the new
milestones/issues, and omits planAhead (→ undefined = badge hidden = in-sync). This is
the durable in-sync state D-07-06 requires (findings #1, #2). The CLI is NOT modified.

IDENTITY MAP (finding #3): the CLI persists new Linear ids to `linear-map.json`
(cli.ts:143, a committed tracked file at repo root → `agenticapps-roadmap/linear-map.json`
in CI). The apply commit MUST include linear-map.json alongside public/roadmap.json, or CI
loses the identity map and risks future duplicate creates.

DIFF CONTRACT (RESOLUTION 1 — typed counts, CLI byte-for-byte unchanged): renderDiff
(diff.ts:168-189) prints ANSI-colored human lines `+ N milestones, + M issues, + L labels`
and `~ D dates (informational only …)`. The workflow (NOT the CLI, NOT the UI) sanitizes:
strip ANSI (`s/\x1b\[[0-9;]*m//g`), parse the two count lines into `{milestones,issues,
labels,dates}`, and emit `___DIFF_JSON___<json>___END_DIFF___`. status.ts (07-02) parses
this typed payload; the UI (07-04) renders the typed object. No UI-side ANSI/regex parsing.

RUN-NAME CONTRACT (RESOLUTION 2 + finding #5): `run-name: backfill [proj:${{ inputs.project }}]
[mode:${{ inputs.mode }}] [cid:${{ inputs.correlation_id }}]`. This is the only channel
status.ts/dispatch.ts can read a run's project/mode/correlation from (the runs API omits
inputs).

SHELL SAFETY (finding #6): `${{ inputs.project }}` is expanded BEFORE the shell parses, so
quoting is not safe. Pass inputs through `env:` (`PROJECT_KEY`, `MODE`) and use
`"$PROJECT_KEY"` / an `if [ "$MODE" = apply ]` shell branch — never `${{ inputs.* }}`
inside a `run:` command string.

CONCURRENCY (finding #9): parallel apply runs and snapshot.yml can race on
roadmap.json/linear-map.json. Put backfill.yml and snapshot.yml in ONE shared serialization
group `roadmap-git-writer` (cancel-in-progress: false); add `git pull --rebase origin main`
before push in backfill's commit step as defense-in-depth.
