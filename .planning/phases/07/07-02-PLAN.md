<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → Pages Function (`/api/backfill/*`) | Untrusted `project`/`mode`/`run` inputs cross here; Access-gated at the edge (D-07-03). |
| Pages Function → GitHub REST API | `GH_BACKFILL_TOKEN` crosses server-side only, in the Authorization header, never in a response body. |
| GitHub Actions runner → sibling private repos | `GH_CROSS_REPO_TOKEN` (fine-grained PAT) clones the 4 named repos. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure | dispatch.ts / status.ts | mitigate | Single try/catch → generic 5xx; token only in `GH_HEADERS` Authorization; tests assert token value + `/ghp_|github_pat_/` absent from every response body (200/400/500/502). |
| T-07-02 | Tampering | dispatch.ts + backfill.yml | mitigate | Reject empty/non-string `project` and invalid `mode` before the GitHub call; quote `inputs.project` as a CLI arg in YAML (no shell interpolation); unmatched project = hard CI failure (CLI zero-match guard, unchanged). |
| T-07-03 | Elevation of Privilege | GH_CROSS_REPO_TOKEN (backfill.yml) | mitigate (Phase-8 binding) | Fine-grained PAT scoped to the 4 named `agenticapps-eu` repos only (Actions: write on roadmap + Contents: read on all 4); NOT org-wide/classic. Binding is Phase 8; documented in 07-05 HUMAN-UAT. |
| T-07-04 | Spoofing | `/api/backfill/*` routes | accept | New routes inherit the existing (not-yet-proven-live) `/api/*` Cloudflare Access policy; no new Access config this phase; proven end-to-end in Phase-8 UAT (mirrors the open 03-ACCESS-PROOF gap). |
| T-07-05 | Denial of Service | dispatch.ts / status.ts | mitigate | Absent `GH_BACKFILL_TOKEN` binding → generic 500 before any fetch (mirrors `!env.LINEAR_API_KEY`), so preview/local builds without the secret fail closed, not crash. |
| T-07-SC | Tampering (supply chain) | npm installs | mitigate | ZERO new npm dependencies added this phase — diff readback uses job-logs-grep, not `fflate`/artifact-unzip. No install task exists, so no `[ASSUMED]` package enters the tree. |
</threat_model>
