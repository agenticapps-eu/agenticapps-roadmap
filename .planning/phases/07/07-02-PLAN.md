PROJECT ALLOW-LIST (server-side, finding #4/#6): the ONLY accepted `project` values are
the three sync.config keys: `["claude-workflow", "cparx", "fx-signal-agent"]`. Any other
value is a 400 before any fetch. (The UI in 07-04 sends the config KEY, resolved from a
Linear-project-id → key map; it never sends a Linear display name.)

RUN-NAME CONTRACT (set by backfill.yml in 07-06; consumed here): every backfill run's
GitHub run-name is `backfill [proj:<project>] [mode:<mode>] [cid:<correlationId>]`. This
is the ONLY channel exposing a run's project/mode/correlation to the runs API (which does
NOT return `inputs`). dispatch.ts (preview verify) and status.ts (correlation match +
identity verify) both parse this run-name.

GitHub REST contracts (RESEARCH Code Examples §1-3, corrected per review):
- Dispatch: POST `${GITHUB_API}/repos/agenticapps-eu/agenticapps-roadmap/actions/workflows/backfill.yml/dispatches`
  body `{ ref: "main", inputs: { project, mode, correlation_id }, return_run_details: true }`.
  200 → `{ workflow_run_id }`; 204 → fall back to `{ runId: null, correlationId }`.
- Get run: GET `${GITHUB_API}/repos/.../actions/runs/${runId}` → `{ id, name, path, head_branch, event, status, conclusion }`.
- List runs (correlation resolve): GET `${GITHUB_API}/repos/.../actions/workflows/backfill.yml/runs?event=workflow_dispatch&per_page=30` → `{ workflow_runs: [...] }`.
- Jobs for run (REQUIRED intermediate — a run id is NOT a job id): GET `${GITHUB_API}/repos/.../actions/runs/${runId}/jobs` → `{ jobs: [{ id, name }] }`.
- Job logs (plain text): GET `${GITHUB_API}/repos/.../actions/jobs/${jobId}/logs`.
- REQUIRED headers on every call (RESEARCH Pitfall 3): Authorization: Bearer, Accept:
  application/vnd.github+json, X-GitHub-Api-Version: 2022-11-28, User-Agent (non-empty).

DIFF MARKER PAYLOAD (RESOLUTION 1 — typed, not raw ANSI text): backfill.yml (07-06)
emits `___DIFF_JSON___<json>___END_DIFF___` where `<json>` is ALREADY the sanitized typed
object `{"milestones":N,"issues":M,"labels":L,"dates":D}` (ANSI-stripped, header-stripped
IN the workflow). status.ts just `JSON.parse`s the captured payload and returns it as
`diff` — NO ANSI stripping, NO regex-parsing of human text on the server.

DIFF READBACK CHOICE: use run→jobs→job-logs grep (zero new npm dependencies). Do NOT add
`fflate` or any artifact/zip path, and do NOT use the run-level `/runs/{id}/logs` endpoint
(it returns a zip) — Simplicity First (RESEARCH "Don't Hand-Roll").

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → Pages Function (`/api/backfill/*`) | Untrusted `project`/`mode`/`previewRunId`/`run`/`correlationId` inputs cross here; Access-gated at the edge (D-07-03). Server-side allow-list + preview-before-apply + run-identity verification enforced in the Function, not only the UI. |
| Pages Function → GitHub REST API | `GH_BACKFILL_TOKEN` crosses server-side only, in the Authorization header, never in a response body. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure | dispatch.ts / status.ts | mitigate | Single try/catch → generic 5xx; token only in `GH_HEADERS` Authorization; tests assert token value + `/ghp_|github_pat_/` absent from every response body (200/400/403/500/502); `Cache-Control: no-store` on every response. |
| T-07-02 | Tampering | dispatch.ts | mitigate | `project` accepted ONLY from the fixed allow-list `["claude-workflow","cparx","fx-signal-agent"]` (finding #4/#6); `mode` enum-validated; both rejected before any fetch. |
| T-07-10 | Elevation of Privilege (approval bypass) | dispatch.ts apply path | mitigate | Apply requires a `previewRunId` that the Function verifies server-side (workflow path, `main`, `workflow_dispatch`, completed, success, run-name `[proj:<same>]`+`[mode:dry-run]`) — a direct `{mode:"apply"}` POST without a real prior successful preview is rejected 403 (finding #5, honors the "printed diff then explicit yes per project" rule server-side). |
